# 开发日志

## 2026-06-30

### v2.0 爬虫系统完整实现

#### 背景

从 v1.0 硬编码 `data_definitions.py` 架构升级为 v2.0 数据驱动架构。新架构下，数据源统一为 JSON 文件（`curated/` 手工审核、`crawled/` 爬虫产出），`data_definitions.py` 降级为纯 JSON 加载器，新增完整爬虫模块实现自动数据收集。

#### 变更内容

**核心重构（`scripts/collector/`）：**
- `data_definitions.py` — 重写为纯 JSON 加载器，从 `curated/*.json` 动态读取 BATCHES
- `collector_helper.py` — 提取 `_process_batch_data()` 和 `_after_process_hook()` 共享逻辑：
  - `_process_batch_data()`: 实体去重、ID 分配、Cypher 生成（原 `generate_batch()` 核心）
  - `_after_process_hook()`: 文件写入、状态更新、CHANGELOG 追加、REPORT 生成
  - `generate_batch_from_data()`: 新增函数，自动分配批次号 + 调用共享流程
  - `append_changelog()`: 新增，实现 CHANGELOG.md 增量记录
  - `skipped_entities` 由 `int` 改为 `list[tuple[str, str, str]]` 记录具体跳过实体
- `run.py` — 新增 `--scraped JSON_PATH` 参数，对接爬虫 JSON 导入

**爬虫模块（`scripts/scraper/`）：**
- `config.py` — 10 个主题配置（street-heroes / cosmic / mutants / villains / magic / mcu-only / avengers-expand / xmen-expand / fantastic-four / eternals），含 Wikipedia 分类/列表/搜索策略
- `fetch_wiki.py` — Wikipedia MediaWiki API 封装（缓存、分类发现、infobox 解析、中英文双语字段映射）
- `fetch_fandom.py` — Fandom Wiki 降级源，结构类似 Wikipedia
- `relationship.py` — 关系推断（team→member, enemy→enemy 双向）、去重
- `output.py` — 爬取结果输出为 `crawled/YYYY-MM-DD_主题.json`
- `discover.py` — CLI 入口，支持 `--theme`、`--random N`、`--merge`、`--list-themes`

**数据目录（`scheduled_data/`）：**
- `curated/001_spider-verse-more-x-men-new-villains.json` — Batch 1 数据迁移
- `curated/002_cosmic-expansion-more-heroes.json` — Batch 2 数据迁移
- `crawled/` — 爬虫产出目录
- `crawled_cache/` — 爬虫缓存目录

**文档：**
- `docs/plan/v2.0_scraper_design.md` — 更新「增量可视」章节，说明 CHANGELOG.md 和 index.json `scraped_runs` 字段
- `CHANGELOG.md` — 新增数据更新日志（每次收集自动追加）

#### 设计要点

- **数据流**: JSON（curated/crawled）→ `generate_batch_from_data()` → Cypher → Neo4j
- **共享核心**: `_process_batch_data()` 和 `_after_process_hook()` 同时服务手工批次和爬虫批次
- **追溯性**: `index.json` 新增 `scraped_runs` 字段，提供 crawled JSON → batch 编号 → 统计的完全追溯
- **增量可视**: `CHANGELOG.md` 持久化每次运行的详细增量（新增/跳过实体、关系数量）
- **爬虫去重**: 爬取前查询已有实体列表，爬取后再次过滤，避免重复导入
- **缓存**: 24h 缓存寿命，减少重复请求

#### 遇到的问题

1. **缩进错误（TabError）**：`_process_batch_data` 编辑时引入了 tab 字符，Python 报错
   - 解决：全局替换 tab 为 4 空格

2. **函数多缩进 4 格**：`_find_props` 到 `generate_batch_from_data` 整段多了一级缩进
   - 解决：批量 de-indent 移除 4 空格

3. **相对导入失败（ImportError）**：scraper 模块用 `from .config import ...` 相对导入，直接运行 `python3 scripts/scraper/discover.py` 时报错
   - 解决：改为绝对导入 `from config import ...`

4. **ModuleNotFoundError: collector_helper**：scraper 模块不自动添加 collector 路径
   - 解决：`sys.path.insert(0, 'scripts/collector')`

5. **最终端到端验证**：合成 JSON → `--scraped` → Cypher 生成 → CHANGELOG 写入 → `scraped_runs` 写入，全部通过
   - 测试后清理了 batch_003、CHANGELOG.md、index.json 恢复原始状态
