# Plan: 重构 scripts/ 目录 + 建立正确格式的定时抓取系统

## Context

项目爬取漫威数据生成 Cypher 导入 Neo4j。原有的数据已在 `fixed/` 中修正格式并导入本地 Neo4j（`.env` 配置 `bolt://127.0.0.1:7687`，数据库 `universe`）。

旧 `scripts/` 根目录脚本生成错误格式（单 `name` 字段、直接 `MERGE (c1)-[:REL]->(t1)` 创建空节点）。需要重构目录结构，新建 `collector/` 存放正确格式的数据生成代码。

## 新目录结构

```
marvel-universe-data/
├── fixed/                      # 已导入 Neo4j 的正确数据（只读参考）
├── scheduled_data/             # [新建] 定时抓取的新数据批次
│   └── index.json              # 状态追踪（最新批次号、已用 ID）
│   docs/
│   └── plan/                   # [新建] 计划文档
├── scripts/
│   ├── import/                 # [保留] 导入 Neo4j 的程序
│   └── collector/              # [新建] 抓取/生成数据代码
│       ├── __init__.py
│       ├── collector_helper.py # 解析、去重、扫描工具函数
│       ├── data_definitions.py # 角色/团队数据定义（含中英文）
│       └── run_batch.py        # CLI 入口，供 Hermes/cron 调用
├── .env                        # 本地 Neo4j 连接配置
```

## 格式规范（以 `fixed/` 为准）

| 节点类型 | 示例 |
|---------|------|
| Character | `MERGE (c984:Character {name_en: "Miles Morales", name: "迈尔斯·莫拉莱斯", real_name: "...", species: "...", first_appearance: "...", abilities: "..."})` |
| Team | `MERGE (t17:Team {name_en: "Midnight Suns", name: "午夜之子", type: "超自然团队"})` |
| Location | `MERGE (l34:Location {name: "Chronopolis", type: "时间之城"})` |
| Movie | `MERGE (m30:Movie {title: "死侍", year: "2016"})` |
| Event | `MERGE (e21:Event {event_name: "秘密战争", year: "2015"})` |
| Item | `MERGE (i215:Item {item_name: "Mysterio's Helmet", type: "装备"})` |
| 关系 | `MATCH (a:Character {name_en: "..."}), (b:Team {name_en: "..."}) MERGE (a)-[:中文关系]->(b);` |

关系标签统一中文：`成员`、`敌人`、`盟友`、`亲属`、`来自`、`使用`、`使者`

## 各模块职责

### collector_helper.py
- `scan_existing_data()` → 扫描 fixed/ + scheduled_data/，返回已有实体字典
- `get_used_var_ids()` → 提取所有已用变量名，计算下一个可用 ID
- `get_batch_state()` / `save_batch_state()` → 读写 scheduled_data/index.json
- `build_match_clause()` → 根据实体类型返回正确的 MATCH 子句
- `validate_cypher()` → 验证生成文件语法
- `write_batch_files()` → 生成 batch_NNN/ 目录下各 .cypher 文件

### data_definitions.py
按批次组织的待收集数据，分批定义角色、团队、关系等。关系用 name_en 引用实体。

### run_batch.py
CLI 入口：`--status` 查看进度，`--batch N` 生成指定批次，`--import` 导入 Neo4j，无参数时自动生成下一批。
