## ✨ 主要特性

### 1. 数据模型

6 种节点类型 + 7 种关系类型（含新增的 `出演`）：

| 关系类型 | 中文标签 | 数量 |
|----------|----------|:----:|
| 敌人 (ENEMY_OF) | 敌人 | 114 |
| 盟友 (ALLY_OF) | 盟友 | 76 |
| 成员 (MEMBER_OF) | 成员 | 72 |
| 出演 (APPEARS_IN) | 出演 | 162 |
| 亲属 (RELATIVE_OF) | 亲属 | 10 |
| 使用 (USES) | 使用 | 7 |
| 来自 (FROM) | 来自 | 7 |
| 使者 (HERALD_OF) | 使者 | 1 |

### 2. 数据来源分层

- **`fixed/`**：手工整理、已审核的核心数据，共 ~228 个节点 + 全部关系
- **`scheduled_data/`**：自动生成的增量数据，目前 2 批共 23 角色 + 1 团队
- **`data_definitions.py`**：可扩展的批次定义，按需添加新角色/关系

### 3. 双数据库同步（★ 新增）

支持云端（Neo4j Aura）为主数据库、本地（Desktop）为副本的双库架构：

```
用户 → run.py → 生成批次文件
               → 连接云端 (成功→导入, 失败→记录 failed)
               → 连接本地 (成功→导入, 失败→记录 failed)
               → 更新 REPORT.md
```

- `scripts/collector/run.py` — **统一入口**：替代 `run_batch.py --import` 的升级版，一次性完成生成→云同步→本地同步
  - 无参数：生成下一批 → 自动同步云 + 本地
  - `--retry`：不生成新批次，只重试未同步内容（**cron 推荐用法**）
  - `--status`：查看同步差异
- `scripts/import/sync_to_local.py` — 独立本地同步工具
- `scheduled_data/index.json` 的 `import_log` 字段记录每个目标（cloud/local）的每个批次和 fixed 文件的入库状态（ok/pending/failed）
- 支持 `--status` 查看差异、`--retry` 自动重试、`--force` 全量重跑

### 4. 幂等安全

所有 Cypher 语句使用 `MATCH...MERGE`，重复执行无害。

## 🚀 快速开始

```bash
# 1. 配置连接（参见 .env.example）
cp .env.example .env       # 填入云端凭证
cp .env.example .env.local # 取消注释本地配置

# 2. 查看当前同步状态
python3 scripts/collector/run.py --status

# 3. 生成下一批 + 双库同步（暂无可生成的新批次）
python3 scripts/collector/run.py

# 4. 仅重试未同步内容（不生成新批次）
python3 scripts/collector/run.py --retry

# 5. 在 Neo4j Browser 中探索
# MATCH (c:Character {name_en: "Spider-Man"})-[:出演]->(m:Movie) RETURN m.title
```

## ⏰ Cron 定时任务

自动模式下推荐使用 `--retry`，只重试未同步内容（不生成新批次，避免 cron 意外触发批量生成）：

```bash
# crontab -e
# 每 4 小时重试未入库的内容（幂等安全）
0 */4 * * * cd /path/to/marvel-universe-data && python3 scripts/collector/run.py --retry >> scheduled_data/sync.log 2>&1
```

如果需要定时生成新批次 + 同步，在 `data_definitions.py` 有未完工批次时使用无参数模式：

```bash
# 每日凌晨生成下一批并同步双库
0 3 * * * cd /path/to/marvel-universe-data && python3 scripts/collector/run.py >> scheduled_data/sync.log 2>&1
```

**注意：** `python3 scripts/collector/run.py` 不等于 `run_batch.py --import`。前者（无参数）会生成新批次 + 同步双库；后者是旧版单一目标导入。新项目统一用 `run.py`。

## 🔧 技术细节

### 解决的问题

1. **电影节点孤立**：fixed/entities_movies.cypher 中 29 部电影没有关联任何人物的出场关系。新增 `relationships_movie_appearances.cypher`（162 条 `出演` 关系）覆盖 26 部主要电影的角色关联。

2. **关系生成 Bug**：`run_batch.py` 的 `_find_props` 函数对 Movie/Event/Item/Location 类型使用错误的 `name` 字段匹配，修复为正确的 `title`/`event_name`/`item_name`/`name`。

3. **双库管理空白**：之前没有云端/本地的同步机制，导入只能连一个目标。新增 import_log 状态跟踪和自动重试。

### 已删除的旧文件

清除了 initial commit 中遗留的旧版独立脚本（`scripts/add_chinese_names.py`、`scripts/expansion.py` 等 13 个文件）和 `marvel_graph_data/` 目录（旧的 GraphML/CSV 中间产物）。
